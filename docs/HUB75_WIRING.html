HUB75 RGB Matrix - Wiring Guide (Phase 1 bring-up)

Bench wiring for the AnimatedPixelClock HUB75 port: a Waveshare ESP32-S3 controller driving 2x Waveshare P2.5 64x64 HUB75E panels chained into a single 128x64 RGB canvas.

Read the whole sheet once before connecting anything. Power-on order and a common ground matter (Section 5). Keep brightness low for the first light.

2. HUB75E connector pinout (16-pin, 2x8 IDC)

Looking at the panel's JIN (input) header. The E line is present because this is a 64x64 / 1/32-scan panel (plain HUB75 would have GND there instead).

   pin  signal        pin  signal
   ---  ------        ---  ------
    1   R1             2   G1
    3   B1             4   GND
    5   R2             6   G2
    7   B2             8   E
    9   A             10   B
   11   C             12   D
   13   CLK           14   LAT (STB)
   15   OE            16   GND
Verify against the Waveshare silkscreen on arrival. Orientation is set by the connector notch / the pin-1 arrow on the PCB. If labels differ, trust the panel silkscreen.

4. Chaining the two panels

  ESP32-S3 --16-pin--> [ Panel 1 ] JOUT --16-pin ribbon--> JIN [ Panel 2 ]
                        x = 0..63                            x = 64..127

ESP ribbon -> panel 1 JIN. Panel 1 JOUT -> panel 2 JIN. Data flows left-to-right (panel 1 = x 0..63, panel 2 = x 64..127). The bring-up seam test (left red / right blue) confirms this; swapped sides = reversed chain order.

5. Power (read the ORDER carefully)

  • Inject 5V to each panel's own power terminals separately (~4A peak per panel; the 10A PSU covers the pair for a mostly-dark clock).
  • Do NOT power panel 2 through panel 1's ribbon - run dedicated 5V/GND wires to each panel.
  • During bring-up: power the controller from USB (flashing + serial), the panels from the PSU.

Power-on order: (1) bond all grounds FIRST - PSU GND <-> panel 1 GND <-> panel 2 GND <-> ESP GND - before any 5V. (2) Apply panel 5V. (3) Plug in the ESP USB last.

Ground-loop caution: the ESP is fed from USB while the panels are fed from the PSU, so their grounds must meet at one point. Never run a panel with its ground floating relative to the ESP. Do not hot-unplug the PSU ground while USB is still attached.

6. Level shifting (only if needed)

Start with direct 3.3V from the S3 (no buffers). With two panels and a short ribbon this is often clean. If you see ghosting / flicker / dim or unstable pixels, buffer the 12 priority lines with the 3x SN74AHCT125N:

  • Buffer (12): CLK, R1, G1, B1, R2, G2, B2, LAT, OE, A, B, C
  • Leave direct on 3.3V (2): D, E (address lines change once per row - safe unbuffered)
  • Per chip: VCC(14)=5V, GND(7)=GND, tie all four ~OE enable inputs LOW (to GND). 3.3V signal into each buffer A input, 5V-level out of its Y output to the panel.

7. Flashing

  • S3-Zero (native USB, no UART chip): hold BOOT/GPIO0 while plugging USB -> download mode -> platformio run -e matrix-s3 --target upload. Serial over native USB-CDC.
  • S3-WROOM-1 devkit (CH343 bridge): just plug the UART USB-C port (auto-reset, no BOOT-hold) -> platformio run -e matrix-s3-wroom --target upload. Serial over UART (platformio device monitor, 115200).

8. FM6126A driver verification (KNOWN UNKNOWN)

The bring-up sketch defaults USE_FM6126A 1 only because Waveshare 64x64 units commonly need it - a guess until checked. Read the IC markings on the panel back. Empirically: if the screen stays blank with init ON, set USE_FM6126A 0 and reflash (and vice-versa). Record the actual chip.

Checklist: grounds bonded first -> panel 5V on -> USB last -> brightness low for first light -> flash -> watch the pattern cycle.
